Introduction

Hospital Management System API Docs

Base URL: http://localhost

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

This API is not authenticated.

PATIENTS :: Store

Example request:
curl --request POST \
    "http://localhost/api/patients/add" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"first_name\": \"bmwswwrfqg\",
    \"last_name\": \"qscrduucmiffaaanipzpsr\",
    \"date_of_birth\": \"2024-06-17T10:05:16\",
    \"gender\": \"sg\",
    \"phone\": \"p\"
}"

const url = new URL(
    "http://localhost/api/patients/add"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "first_name": "bmwswwrfqg",
    "last_name": "qscrduucmiffaaanipzpsr",
    "date_of_birth": "2024-06-17T10:05:16",
    "gender": "sg",
    "phone": "p"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),

POST api/patients/add

Headers

Content-Type      

Example: application/json

Accept      

Example: application/json

Body Parameters

first_name   string   

'user_id' => 'nullable|string|max:255|unique:people', 'person_type_id' => 'required|integer|exists:person_types,id', 'city_id' => 'required|integer|exists:cities,id',. Must not be greater than 100 characters. Example: bmwswwrfqg

last_name   string   

Must not be greater than 100 characters. Example: qscrduucmiffaaanipzpsr

date_of_birth   string  optional  

Must be a valid date. Example: 2024-06-17T10:05:16

gender   string  optional  

Must not be greater than 10 characters. Example: sg

phone   string  optional  

Must not be greater than 20 characters. Example: p
